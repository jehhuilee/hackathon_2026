# AI 모의면접 코칭 — 실전 맥락 기반 멀티모달 피드백

> 발표 흐름: ① 불만 수집 → ② 문제 정의 → ③ 우리가 해결한 것 → ④ 방법론 → ⑤ 결과물(데모)

---

## 1. 커뮤니티의 AI 면접(모의면접·코칭) 불만 수집

채용용 "AI 면접"이 아니라 **AI 모의면접 · 면접 코칭 · 피드백 서비스**에 대한 실제 사용자 불만(Reddit, G2, Trustpilot, KCI 논문 등)을 수집했다. 핵심을 압축하면:

- **피드백이 너무 일반적이다** — "좋아요 / 더 자신감 있게" 수준이라 실전 개선에 도움이 안 됨
- **직무·회사·레벨 맥락이 약하다** — 개발·기획·마케팅·시니어마다 평가 기준이 달라야 하는데 일반 코칭에 머묾
- **실제 면접을 재현하지 못한다** — 녹화+리포트에 가까워 꼬리질문·압박감 같은 실전감이 없음
- **전달 방식(말투·속도·필러워드)에 치우친다** — 정작 답변의 논리·설득력·내용 평가는 약함
- **개선 행동으로 연결되지 않는다** — 시선·억양을 잡아줘도 "그래서 어떻게 고치냐"가 모호함
- **신뢰·개인정보 불안** — 채점을 안 해줘 답답하고, 녹화·음성 데이터 저장/삭제가 불투명함

### VOC 한 줄 요약
> "AI가 말 빠르기·필러워드·시선은 잡아주는데, **내 답변이 왜 설득력이 없는지**는 못 짚어주고, **실전(꼬리질문·압박)**과 거리가 멀며, 피드백이 **직무 맥락 없이 너무 일반적**이다."

---

## 2. 문제 정의

위 불만은 결국 **세 가지 갭(gap)** 으로 수렴한다.

| # | 문제 | 본질 |
|---|------|------|
| P1 | **얕은 피드백** | 점수/칭찬은 주지만 "문장 단위로 무엇이 부족한지"를 못 짚음 |
| P2 | **맥락 부재** | 직무·회사·이력서·면접 난이도가 질문과 채점에 반영되지 않음 |
| P3 | **전달 방식 편중 & 비실전성** | 음성/자세 분석은 하지만 답변 내용 평가, 꼬리질문, 압박 재현이 빠짐 |

→ 한마디로: **"내용까지 평가하고, 실전 맥락을 재현하며, 바로 고칠 수 있는 피드백을 주는 AI 모의면접"** 이 없다.

---

## 3. 우리는 여기서 이걸 해결했다

각 문제를 코드로 구현한 기능과 직접 연결했다.

### ✅ P1 해결 — "얕은 피드백"을 **수정 가능한 답변 단위 피드백**으로
- 답변마다 LLM이 **종합/내용/구조 점수 + 강점 + 약점 + 개선점 + 모범답안**을 생성
- 평가 기준을 명시: 질문 적합성 · 직무 관련성 · 논리성 · 구체성 · 경험/역할의 명확성 · STAR 기법
- "면접에서 바로 개선할 수 있는 방식으로 제안" 하도록 설계 → 칭찬이 아니라 **고칠 거리**를 반환
- 결과는 STT 전사(녹음 답변 → 텍스트)와 함께 리포트에 축적

### ✅ P2 해결 — **직무·이력서·면접관 페르소나**를 전 과정에 주입
- 질문 생성이 **지원 직무·회사·기술스택·이력서 텍스트** 기반 → 일반 질문이 아닌 맞춤 질문
- 직무 인지형 설계: 개발 직군이 아니면 기술 깊이를 억지로 요구하지 않음
- **면접관 3종 페르소나(난이도 A/B/C)** 도입 — 같은 이력서라도 난이도에 따라 질문 깊이·압박 강도·채점 strictness가 달라짐
  - A 친근한(스타트업) / B 표준(중견·대기업) / C 엄격한(탑티어·외국계)

### ✅ P3 해결 — **멀티모달 + 내용 평가 + 실전 재현**을 한 세션에
- **실시간 음성 코칭**: 말 빠르기(초당 음절)·피치 떨림·침묵을 0.5초 주기로 분석해 즉시 경고(TOO_FAST / PITCH_UNSTABLE / LONG_SILENCE)
- **실시간 자세/표정 코칭**: 얼굴 감지·상체 움직임·손동작·어깨 기울기를 브라우저에서 실시간 분석하고 교정 문구 제공
- **내용 평가 병행**: 전달 방식만이 아니라 답변 텍스트의 논리·구체성까지 LLM이 채점 → "말투 편중" 해소
- **꼬리질문(실전 재현)**: 엄격한 면접관(C)에서는 답변 키워드를 짚는 꼬리질문을 자동 생성 → 압박·후속질문 상황 재현

### 🔑 핵심 차별점 — **하나의 "면접관 난이도"가 전 모듈을 관통**
같은 A/B/C 선택이 **① 질문 생성 깊이 ② 답변 채점 강도 ③ 꼬리질문 활성화 ④ 실시간 음성 경고 임계값**에 일관되게 적용된다. 엄격한 면접관을 고르면 질문도 날카로워지고, 채점도 짜지고, 같은 발화라도 음성 경고가 더 빡빡하게 뜬다. → "면접 난이도"라는 하나의 축으로 **실전감의 강약을 통째로 조절**하는 것이 우리 제품의 정체성.

### 🔒 신뢰·프라이버시 보강 (부가)
- 실시간 음성 분석은 **외부 API 없이 100% 로컬 DSP** 로 동작
- STT는 **로컬(faster-whisper) / 클라우드(OpenAI)** 백엔드를 환경변수로 선택 → 민감 데이터 로컬 처리 가능
- 음성 DSP 의존성이 없어도 REST·평가 기능은 계속 동작하도록 **그레이스풀 디그레이드**, WebSocket 연결 실패도 사용자에게 노출되도록 처리

## 4. 간단한 방법론

**Two-Track 멀티모달 파이프라인** — 실시간성과 정확도를 분리해서 둘 다 잡는다.

```
                    ┌──────────────── 실시간 트랙 (저지연, 로컬) ─────────────────┐
   마이크/카메라 ──► │  음성: WebSocket → 링버퍼 → DSP 분석 → 0.5s 경고          │ ──► 라이브 코칭 UI
                    │  영상: MediaPipe 포즈 → 자세/표정 지표 → 실시간 교정 문구   │
                    └────────────────────────────────────────────────────────────┘
                    ┌──────────────── 사후 트랙 (정확도 우선) ───────────────────┐
   답변 종료 ──────► │  녹음 → STT 전사 → LLM 평가(점수+개선점+모범답안)          │ ──► 답변 결과 + 리포트
                    │  (엄격 모드: 꼬리질문 생성)                                  │
                    └────────────────────────────────────────────────────────────┘

   ▲ 면접관 페르소나(A/B/C)가 두 트랙의 임계값·질문 깊이·채점 강도에 동시 주입
```

- **실시간 트랙**: 0.5초 이내 피드백을 목표로, 무거운 분석은 사후로 분리해 지연을 통제
- **사후 트랙**: 답변 전체를 한 번에 전사·평가해 정확도 확보
- **페르소나 일관 적용**: 설정에서 고른 난이도 하나가 모든 모듈로 전달되어 일관된 면접 경험을 만듦
- **모듈 분리 + 통합 게이트웨이**: 음성(Audio)·영상(Motion)·언어(LLM)·STT를 독립 모듈로 두고 단일 FastAPI 게이트웨이로 통합

## 5. 구현의 결과물 (데모 시나리오)

### 사용자 흐름
1. **설정** — 직무·회사·기술스택·이력서 입력 + **면접관 유형(A/B/C) 선택** → 맞춤 질문 자동 생성
2. **면접 진행** — 질문별로 영상 녹화 중 **실시간 피드백** 동시 표시
   - 🔊 음성: "말이 빨라지고 있어요" / "목소리 떨림이 큽니다" / "침묵이 길어지고 있어요"
   - 🧍 자세: "화면 중앙을 바라보세요" / "상체 움직임이 큽니다" / "어깨를 정면으로"
   - 🎯 선택한 면접관 강도가 상단에 표시되고, 그 강도대로 경고 빈도가 달라짐
3. **답변 종료 → 즉시 분석** — 전사 텍스트 + 종합/내용/구조 점수 + 강점·약점·개선점 + 모범답안
4. **최종 리포트** — 질문별 점수, 평균 점수, 전사, 음성 지표, 자세 지표를 한눈에 → 반복 연습 시 성장 추적

### 데모에서 보여줄 "킬러 장면"
- **같은 답변, 다른 면접관**: A로 한 번, C로 한 번 → 질문 난도·채점 점수·실시간 음성 경고 빈도가 눈에 띄게 달라지는 것을 나란히 시연
- **실전 재현**: C 모드에서 답변 직후 **꼬리질문**이 따라붙는 장면
- **수정 가능한 피드백**: "더 자신감 있게"가 아니라 "성과 수치가 없음 → 모범답안은 이렇게" 로 구체적으로 떨어지는 장면

### VOC → 결과물 매핑 (요약)
| 사용자 불만 | 우리 결과물 |
|------------|------------|
| 피드백이 너무 일반적 | 답변 단위 점수 + 약점 + 개선점 + 모범답안 |
| 직무·레벨 맥락 부족 | 직무/이력서 기반 질문 + 면접관 난이도 A/B/C |
| 실제 면접과 다름 | 엄격 모드 꼬리질문 + 페르소나별 압박 강도 |
| 전달 방식에 치우침 | 음성·자세 실시간 + 답변 내용 LLM 평가 병행 |
| 개선 행동으로 연결 안 됨 | 즉시 경고 + 개선점·모범답안 + 누적 리포트 |
| 개인정보·신뢰 불안 | 로컬 DSP·로컬 STT 옵션, 외부 API 없이 동작 가능 |
